UI:
Пользователь выбирает нужный пункт, в случае некорректного ввода возвращается в главное меню.
Алгоритм:
Мы делим текст из входного потока на N равных части(насколько это возможно), отправляем эти части на обработку N потокам (N указывает пользователь). После завершения потока они синхронизируют данные в словаре(Dictionary) и выводится результат.
Для реализации многопоточности решил использовать стандарную System.Threading, не использовал openMP и MPI, хоть они и более эффективные, т.к. в описании задания небыло никакой конкретики про многопоточность и сторонние библиотеки
